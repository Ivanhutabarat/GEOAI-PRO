## 🛡️ 2. Security & Infrastructure Hardened Systems

<details>
<summary><b>Click to expand System Security & Hardening Protocols</b></summary>

*   **🛡️ Hardware License & Integrity Lock (`identityValidator`)**:
    The system utilizes a physical binary signature anchor (`/config/.identity_lock`). React is gated behind a `prebootIntegrityCheck()` that validates a SHA-256 checksum server-side before rendering the UI.
*   **🔒 Global Sanitization & Masking Engine**:
    All outgoing telemetry, logs, and server responses pass through a Regex-based scrubbing pipeline that intercepts sensitive signals, phone numbers, and authorized emails, replacing them with secure hashes to prevent leakage.
*   **⚠️ 4-Tier WhatsApp OTP Shield**:
    Administrative destructive actions (complete purging of models) are isolated behind a 4-tier validation channel linked via `process.env.OWNER_WHATSAPP` (`+6285260245100`).
*   **⚙️ Bulletproof Vite-Express Middleware**:
    To prevent server-side regressions, the system employs a custom HTML transformation middleware (`server.ts`) that guarantees the mandatory React Fast Refresh Preamble is always injected during dynamic HTML responses.
</details>

## 🧪 6. Van-Botz WhatsApp Integration

The GeoAI Pro workstation includes a deep-integration WhatsApp bridge derived from the `Van-Botz` architecture principles.

<details>
<summary><b>Click to expand Bot Control Features</b></summary>
The **WhatsAppBotMenu** (bottom-right toggle) provides direct control over the telemetry bridge:
- **🗂️ Generate & Send Report**: Automatically compiles current session data into a PDF and dispatches it to the secure admin number (`6285260245100`).
- **📢 Broadcast Status**: Broadcasts heartbeat and system status signals from the GeoAI Pro core to the registered WhatsApp network.
- **⚙️ System Status**: Directly queries the internal bot-bridge status and authentication telemetry.
